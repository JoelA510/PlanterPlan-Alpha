## Step 2: SIMPLIFY ‚Äî Reduce complexity in remaining code

### 2a. Remove trivial `useCallback` wrappers
**`src/features/tasks/components/TaskList.tsx` (lines 64-66)**:
```ts
const fetchTasks = useCallback(() => refetchProjects(), [refetchProjects]);
const fetchProjectDetails = useCallback(() => refetchProjects(), [refetchProjects]);
const refreshProjectDetails = useCallback(() => refetchProjects(), [refetchProjects]);
```
Three identity wrappers ‚Äî just use `refetchProjects` directly.

**`src/features/tasks/components/TaskControlButtons.tsx`** and **`TaskStatusSelect.tsx`**: Multiple `useCallback` wrappers around trivial event handlers where dependencies (`task` prop) change every render anyway. Inline these handlers.

### 2b. Flatten unnecessary type aliases
`src/shared/db/app.types.ts` defines:
```ts
export type Task = TaskRow;
export type Project = TaskRow;
export type ProjectRow = TaskRow;
```
Three aliases for the same type create semantic confusion. Consolidate to `TaskRow` as canonical and `Project` as the only alias.

### 2c. Consolidate duplicate entity clients
`Phase` and `Milestone` entity clients in `src/shared/api/planterClient.ts` are identical `createEntityClient` calls with no overrides. Merge or alias.

### 2d. Inline `createPageUrl` utility
`src/shared/lib/utils.ts` exports `createPageUrl()` used 14 times ‚Äî it just lowercases and prepends `/`. Replace all call sites with string literals (e.g., `"/dashboard"`).

## Step 3: ACCELERATE ‚Äî Fix performance issues

### 3a. Add React Query cache configuration
`src/features/tasks/hooks/useTaskQuery.ts` ‚Äî All queries use default `staleTime`/`gcTime`. Add explicit cache strategies to prevent unnecessary refetches.

### 3b. Memoize inline filter/map in TaskList
`src/features/tasks/components/TaskList.tsx` (lines 74-76):
```ts
tasks.filter((t: TaskRow) => t.origin === 'instance')
tasks.filter((t: TaskRow) => t.origin === 'template')
```
Created every render without `useMemo`. Wrap with `[tasks]` dependency.

### 3c. Batch parent date updates
`src/features/tasks/hooks/useTaskActions.ts` (lines 255-259) calls `updateParentDates()` up to 3 times sequentially after a single task update. Batch into a single call or RPC.

### 3d. Reduce query count in useProjectData
`src/features/projects/hooks/useProjectData.ts` makes 3 separate queries (project, hierarchy, team). Use existing `getWithStats()` method to eliminate 1 query.

### 3e. Lazy-load heavy dependencies
Add dynamic imports in `vite.config.ts` for:
- `recharts` (~3.7MB) ‚Äî only used on Reports page
- `framer-motion` (~12MB) ‚Äî only used in 3 components

### 3f. Replace `useDeepCompareMemo` with `useMemo`
`src/features/tasks/hooks/useTaskDragAndDrop.ts` (line 35) ‚Äî expensive deep comparison on objects that change frequently. Switch to shallow `useMemo`.

---

## Step 4: Fix build-breaking issues

### 4a. Fix missing `useCallback` import
`src/features/tasks/components/TaskList.tsx` ‚Äî uses `useCallback` (lines 64, 65, 66, 180) but never imports it. Runtime crash.

### 4b. Fix raw Supabase call bypassing API layer
`src/features/tasks/hooks/useTaskQuery.ts` (lines 43-51) ‚Äî Template query uses raw `supabase.from()` instead of `planter.entities.Task`. Inconsistent with rest of codebase.
